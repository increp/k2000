# Franklin v1.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Version:** 5.20 (artifact) · **Spec:** `docs/superpowers/specs/2026-07-05-franklin-v1.1-design.md` (v5.19) · **Branch:** `feat/franklin-v1.1` (exists; spec committed)

**Goal:** Fix the repaint-killed dropdowns structurally, give every test a six-field info card with run provenance, add a searchable catalog browser, and keep the run schema open for future capture/comparison kinds.

**Architecture:** Franklin tab moves from one monolithic repaint to five persistent section containers with per-section renderers and data-change/interaction guards. Provenance is detected in the C++ producer (env priority chain), stamped into `start` events, surfaced per run and per card. Catalog goes to v2 (two new required prose fields, full 291 backfill) with the drift rule extended.

**Tech Stack:** C++17/JUCE (producer) · Node 24 native TS, node:test, esbuild, no new deps (dashboard) · Python stdlib (drift-check).

## Global Constraints

- Builds ALWAYS `-j4`; suite: `cmake --build build --target k2000_tests -j4 && ./build/tests/k2000_tests | tee build/last-test-run.log` → **stays `Summary: 291 tests, 0 failed`** (new RunLog assertions go INSIDE the existing single `beginTest` section — no count bump, no README change).
- Runner detection priority (exact): `BERNIE_RUNNER` env (verbatim string) → `GITHUB_ACTIONS` set → `"ci"` → `CLAUDECODE` set → `"claude"` → `"terminal"`. Dashboard spawns add `BERNIE_RUNNER=dashboard`.
- Display strings (exact): ci → `CI` · claude → `Claude` · dashboard → `you (dashboard)` · terminal → `you (terminal)` · absent → `unknown` · any other value → verbatim.
- Catalog v2: entries add REQUIRED `compares` and `succeedingMeans`; `version: 2`; UI tolerates v1 (missing fields render as em-dash); drift rule WARNs on any missing prose field. No filler — same audited bar as v1.
- Unknown run `kind` values are parsed, listed, and rendered generically (neutral badge, verbatim label) — never dropped (maturity floor for future `capture`/`comparison`).
- Dashboard rails: no new npm deps; `.ts` import extensions; `import type` only for server modules from `src/`; every user-derived string through `esc`/`escAttr`; localhost bind unchanged.
- Archives immutable: no migration of old runlogs; missing `runner` renders `unknown`.

function paintSection(name: string, html: string): void {
