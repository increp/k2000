# Plans

Implementation plans for the specs in [`../specs/`](../specs/). A plan turns a spec into an ordered sequence of bite-sized tasks an engineer can execute. Each task is TDD-shaped where possible: write the failing test, watch it fail, implement, watch it pass, commit.

Filename convention: `YYYY-MM-DD-<feature>-implementation.md`. The date is when the plan was written.
