# v4.5(B) — Summit UI foundation

**Status:** Implemented — shipped as v4.5.0 (2026-06-16).

**Scope:** Replace the utilitarian editor with an extensible, Summit-aesthetic **UI foundation** — a reusable component/theme framework plus the Summit-faithful panel layout, populating only the sections whose engine exists today and reserving space for the rest. Load-bearing for the engine roadmap: it lands **before v5**, and every later phase ships its feature UI on top of it. Vector/custom-drawn (photoreal art is v12+); the full graph/wiring editor is v6.

**Context:** Part of the v4.5 milestone (A: KB OCR fix ✅; C: K2061 re-positioning ✅). Grounds the GUI principle from the C re-positioning ([ADR 0010](../decisions/0010-k2061-repositioning-constant-summit-spine.md), [engine register](../architecture/engine-questions.md) Q9): the constant Summit spine is a permanent panel; the variable K2061 source/DSP region is a dynamic, swappable workspace.

## Locked decisions (this brainstorm)

- **Layout direction A — Summit hardware-faithful**, left → right subtractive flow, with **proportional sizing**: the VAST source/DSP region is the large workspace; the constant spine modules (filter especially — a handful of knobs) are compact.
- **Build framework + live sections only**, layout **reserves zones** for future sections so the panel never reflows as phases land.

## Layout

```
┌───────────────────────────────────────────────────────────────────┐
│  MASTER · PRESET · ACTIVE VOICES                                    │
├───────────────────────────────────────────────────────────────────┤
│  VAST SOURCE / DSP  (large, swappable)        │MIX│  FILTER  │DRV│AMP│
│  [ OSC A ] [ ALGORITHM ] [ SHAPER ]  …        │   │ (Huggett)│   │   │
├───────────────────────────────────────────────────────────────────┤
│  AMP ENV     │  MOD ENVS*  │  LFO 1–4*  │  MOD MATRIX*  │  FX*       │
└───────────────────────────────────────────────────────────────────┘
   * reserved zone — drawn, framed, dimmed; lit up at its engine phase (v5–v8)
```

- **Live now** (wired to v4 params): the **source/DSP region** (osc waveform/coarse/fine · algorithm selector · shaper drive/mix as clusters), **FILTER** (v4 SVF — becomes the Huggett spine filter at v5), **AMP** (ADSR), **MASTER** (gain), and the **per-layer routing** (the edit-layer selector + enable/key/vel/channel/level, folded in from v4's routing strip).
- **Reserved** (framed, inert, dimmed): MOD ENVS, LFO 1–4, MOD MATRIX, FX CHAINS. The MIXER and DRIVE slots are placeholders too until v5/v7.

## Architecture

The foundation is a small set of reusable units; the editor composes them.

| Unit | Responsibility |
|---|---|
| `SummitLookAndFeel` (`LookAndFeel_V4`) | The single source of the dark Summit aesthetic — knob rendering, colors, fonts, section framing. Swap it and the whole panel re-themes. |
| `LabeledKnob` | A rotary control + value readout + caption; the atomic Summit control. |
| `Section` | A titled, bordered Summit "module" that lays out a group of controls; supports a **reserved/dimmed** state for not-yet-built sections. |
| `ParamBinder` | Owns APVTS `SliderAttachment`/`ComboBoxAttachment`/`ButtonAttachment` with a **detach-before-rebind** contract — formalizing the v4 fix so no future per-layer/per-block re-point can corrupt the param you left. |
| `K2000AudioProcessorEditor` | Composes `Section`s into the proportional-A grid; owns the `SummitLookAndFeel` and the `ParamBinder`s; replaces today's utilitarian editor. |

**Dynamic source region.** The source/DSP region holds the active blocks as **swappable knob-clusters**, (re)bound through `ParamBinder`. This is the *seed* of v6's graph/wiring editor — B establishes the region and the swap pattern (cluster-per-block, detach-before-rebind), not the wiring canvas. The edit-layer selector re-points the whole panel to the chosen layer's params via `ParamBinder` (the v4 behavior, now framework-guaranteed safe).

## Testing

- The editor has no unit harness (`K2000_TESTING=1` makes `createEditor()` return null), so the panel itself is verified by **build + manual Ableton smoke**, as before.
- **New, and directly retiring the v4 editor-test-gap concern:** a small **headless `ParamBinder` test** — attach a control to param X, rebind to param Y, assert X is untouched (the exact class of bug behind v4's layer-switch corruption). `ParamBinder` is plain enough to test without a live editor.
- Manual smoke checklist: every live control drives its param; the edit-layer selector preserves each layer's settings; reserved zones render dimmed/inert; window sizes sanely; a v4 preset loads unchanged.

## Deferred

- **Photoreal art / final polish** — v12+ (this is clean vector, not images).
- **The full graph/wiring editor** for the source/DSP region — v6.
- **Real spine modules + their controls** (Huggett filter, mod envelopes, LFOs, mod matrix, FX) — light up at v5–v8 in their reserved zones.
- **The growing per-block cluster library** — one cluster per new source/DSP block, added with each block's phase.
