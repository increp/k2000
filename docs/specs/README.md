# Specs

Design specifications for major pieces of work. Each spec is the agreed-upon plan before implementation begins; it captures what we're building, why, and what's intentionally out of scope.

Filename convention: `YYYY-MM-DD-<topic>-design.md`. The date is when the spec was written, not when work started.
